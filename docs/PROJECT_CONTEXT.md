## 5. 主项目方向：FlowRAG

项目名：FlowRAG

项目定位：

一个可异步入库、可检索、可流式问答、可引用回溯、可评测、可部署压测的 RAG 后端工程系统。

一句话描述：

面向企业知识库场景设计的 AI 后端系统，支持文档异步入库、向量检索、流式生成、引用回溯、检索评测与工程化部署。

暂定技术栈：

- FastAPI：主后端框架；
- MySQL：存用户、知识库、文档、任务、对话、评测等业务数据；
- Redis：用于缓存、限流、任务状态、Celery broker；
- Celery：用于文档异步入库任务；
- Qdrant：向量数据库；
- Docker Compose：本地和服务器部署；
- Nginx：反向代理；
- LLMProvider：封装第三方大模型 API；
- EmbeddingProvider：封装 embedding 模型或 API。

## 6. 学习方式

不先脱离项目去刷一堆视频，也不直接让 Codex 写完整项目。

采用：

> 项目驱动学习 + 小实验预备营 + 再合入主项目

也就是说：

1. 每次只学一个当前项目马上会用到的知识点；
2. 每个知识点先做一个最小实验；
3. 用户理解后，再逐步合入 FlowRAG 主项目；
4. Codex 要讲为什么这么设计，而不是只给代码；
5. Codex 要控制节奏，不能一下子把 FastAPI、MySQL、Redis、Celery、Qdrant 全塞给用户；
6. 每一阶段结束后，要给用户检查问题，确认用户真的懂了；
7. 如果用户回答不上来，要继续解释，不要急着推进下一阶段。
8. 在进入 Celery 和流式接口前，要先安排一个独立的“FastAPI 异步入门”阶段，先把同步 / 异步 / 阻塞 / 协程这些基础概念讲清楚。
9. 在“FastAPI 异步入门”阶段结束后，还要补一个短的“并发选型与线程/进程最小模板专题”，帮助用户建立 `async / 多线程 / 多进程 / Celery` 的选型能力，并掌握最小模板级写法；该专题不是独立大阶段。

每个小阶段包含：

1. 知识点讲解；
2. 结合代码或最小例题讲解；
3. 用户提问；
4. Codex 先创建作业目录/文件后，再布置小节动手题；
5. 口头提问；
6. 综合课后动手题；
7. 阶段检查提问。

完整课堂顺序和触发时机以根目录 `AGENTS.md` 中“最高优先级：课堂规则”为准。
